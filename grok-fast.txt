GitHub Copilot: The check_docs.py file is a utility script designed to automatically verify that all required docstrings (module, class, and function docstrings) are present in your Python codebase. Here's where and how it's used:

## Purpose and Integration
- **Automated Testing**: It's part of the testing framework you requested for verifying that missing documentation has been properly added.
- **Execution Flow**: 
  1. auto_test.py detects your environment (Windows/Linux/Docker)
  2. It runs the appropriate test script (run_test.bat on Windows, run_test.sh on Linux/macOS)
  3. Those scripts execute `python check_docs.py`
  4. The output is captured, timestamped, and saved to test_run.log

## How to Use It Directly
You can run it manually from the command line in your project directory:

```bash
python check_docs.py
```

It will:
- Parse all Python files in the project (api.py, app.py, config.py, tasks.py, utils.py)
- Check for missing docstrings using Python's AST (Abstract Syntax Tree)
- Print any missing docstrings and exit with code 1 if issues found
- Print "TEST PASSED" and exit with code 0 if all docstrings are present

## Output Example
When run, it produces output like:
```
TEST PASSED
```
Or if missing:
```
Module docstring missing in api.py
Function create_task missing docstring in api.py
TEST FAILED
```

This script ensures code quality by enforcing documentation standards across your project.
